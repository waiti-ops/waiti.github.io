linux系统的简单使用

发表于 2019-06-02

CentOS系统 root 123456 qyz 123456
虚拟机=电脑
快照=备份
创建克隆-创建完整克隆
linux操作系统，适合服务器，不适合办公 稳定 安全性高 免费

1.系统路径
/表示根目录
~表示/root
etc 存放系统配置目录
home 除了root以外所有用户默认在home下新建一个以用户名作为文件夹名称的文件夹
(新建一个用户就会在home下新建一个文件夹)
用户xxx对/home/xxx具有完全操作权限，对其他系统文件夹没有操作权限
root 用户的单独文件夹
usr所有用户安装的软件都放在这个文件夹
在/usr/local下新建tmp文件夹，所有压缩包上传到这个文件夹

2.常用命令
pwd 打印工作目录，当前所有文件夹的的全路径
cd .. 向上跳一级文件夹
cd 路径 进入指定文件夹
mkdir 新建空文件夹
ls 平铺
ll 详细列表
vi 编辑文件，vi————>insert/i————>编辑内容————>esc————>:wq(保存并退出)//:q!强制退出
vim 编辑文件(如果文件不存在带有创建文件功能)带有颜色的编辑
touch 创建空文件
cat 查看文件全部内容
head [-n] 文件名 查看文件的前n行，默认前10行
tail [-n] 文件名 查看文件的后n行，默认后10行
tailf 动态显示文件后n行内容 常用在显示tomcat日志文件功能
echo ‘内容’ >> 文件名 向文件中添加一些内容
ifconfig 打印网卡信息
reboot 重启
tar zxvf 文件名
cp [-r]原文件 新文件路径 :复制文件
-r 复制文件夹
ctrl+c 中断
mv 原文件 新文件 :剪切(具备重命名的功能)
rm [-rf] 文件名 :删除
-r 删除目录
-f 强制删除，不需要确认
clear 清屏

3.linux所有需要写路径的地方都支持两种写法
全路径 以/开头
相对路径 从当前资源一步一步寻找其他资源

4.Xshell
linux的客户端工具
CentOS默认只开启22端口，其他端口都被防火墙拦截

5.Filezilla工具使用
FTP协议
file transfer protocol
作用：互联网中，计算机之间直接传输文件需要遵循的协议
filezilla
基于FTP协议的客户端工具
使用filezilla可以快速完成文件的传输功能

6.安装jdk
把压缩包上传到服务器/usr/local/tmp
解压压缩包

    #tar zxvf 文件名
把解压后的文件复制到/usr/local/jdk7
    #cp -r 文件名 /usr/local/jdk7
配置环境变量，执行命令
    #vim /etc/profile
    修改文件中内容
    (分割使用的是:)
    ($变量名是引用的意思)
    (export 添加或者修改系统变量)
{
#export PATH USER LOGNAME MAIL HOSTNAME HISTSIZE HISTCONTROL
 export JAVA_HOME=/usr/local/jdk7
 export PATH=$JAVA_HOME/bin:$PATH
 export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
 }
解析文件或重启系统
    #source /etc/profile
验证配置是否成功
    #java -version

7.Tomcat配置(linux中绿色表示可执行文件)
把tomcat压缩包上传到服务器
解压tomcat 

    #tar zxvf
解压后的文件夹复制到/usr/local/tomcat7
    #cp -r apache-tomcat-7.0.94 /usr/local/tomcat7
配置环境变量，执行命令
    #vim /etc/profile
    修改文件中内容
    {
    #export TOMCAT_HOME=/usr/local/tomcat7
    export CATALINA_HOME=/usr/local/tomcat7
    }
放行8080端口
    #vim /etc/sysconfig/iptables
    把包含22行的内容复制一份改端口为8080(8080:9000 从8080到9000全放行)
    重启服务
    #service iptables restart (start restart stop)
启动tomcat,进入tomcat7/bin文件夹
    直接启动
        #./startup.sh
    启动并动态打印启动信息
        #./startup.sh & tailf /usr/local/tomcat7/logs/catalina.out

7.mysql安装
上传解压
复制到/usr/local/mysql

    #cp -r mysql-5.6.31 /usr/local/mysql
进入mysql文件夹
    #cd /usr/local/mysql
添加用户组，命名为mysql
    #groupadd mysql
创建用户mysql，并指定所属群组mysql
    #useradd -r -g mysql mysql
变更mysql用户组有操作当前文件夹的权限(必须保证在/usr/local/mysql目录下)
    #chgrp -R mysql .
变更mysql用户有操作当前文件夹的权限
    #chown -R mysql .
初始化
判断/etc/my.cnf是否存在，存在就删除
    #ls /etc/my.cnf
    #rm /etc/my.cnf    
初始化数据库
    #./scripts/mysql_install_db --user=mysql 
修改配置文件
    复制my.cnf
    #cp support-files/my-default.cnf /etc/my.cnf 
    复制启动文件
    #cp support-files/mysql.server /etc/rc.d/init.d/mysql 
如果提示没有mysql命令，需要添加软连接
    #ln -s /usr/local/mysql/bin/mysql /usr/bin/mysql 
启动数据库
    #service mysql start 
操作数据库
    #mysql -u root -p

8.忘记mysql root密码的修改方式和修改访问host
进入/etc/my.cnf在[mysql]下添加skip-grant-tables启动安全模式

    #vi /etc/my.cnf
重启服务
    service mysql restart
登录mysql，输入密码时直接回车
    #mysql -u root -p
进入mysql后，使用mysql数据库
    #use mysql
修改密码
    #update user set password= password("123456") where user='root';
刷新权限
    #flush privileges;
退出mysql编辑模式
    #exit
修改访问host
    #use mysql
    #grant all privileges on *.* to root@'%' identified by '123456' with grant option;
    #flush privileges;
    #exit 
    #vim /etc/sysconfig/iptables
    复制22端口为3306
    #service iptables restart
