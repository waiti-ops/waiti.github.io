linux系统的简单使用

发表于 2019-06-02

CentOS系统 root 123456 qyz 123456
虚拟机=电脑
快照=备份
创建克隆-创建完整克隆
linux操作系统，适合服务器，不适合办公 稳定 安全性高 免费

1.系统路径
/表示根目录
~表示/root
etc 存放系统配置目录
home 除了root以外所有用户默认在home下新建一个以用户名作为文件夹名称的文件夹
(新建一个用户就会在home下新建一个文件夹)
用户xxx对/home/xxx具有完全操作权限，对其他系统文件夹没有操作权限
root 用户的单独文件夹
usr所有用户安装的软件都放在这个文件夹
在/usr/local下新建tmp文件夹，所有压缩包上传到这个文件夹

2.常用命令
pwd 打印工作目录，当前所有文件夹的的全路径
cd .. 向上跳一级文件夹
cd 路径 进入指定文件夹
mkdir 新建空文件夹
ls 平铺
ll 详细列表
vi 编辑文件，vi————>insert/i————>编辑内容————>esc————>:wq(保存并退出)//:q!强制退出
vim 编辑文件(如果文件不存在带有创建文件功能)带有颜色的编辑
touch 创建空文件
cat 查看文件全部内容
head [-n] 文件名 查看文件的前n行，默认前10行
tail [-n] 文件名 查看文件的后n行，默认后10行
tailf 动态显示文件后n行内容 常用在显示tomcat日志文件功能
echo ‘内容’ >> 文件名 向文件中添加一些内容
ifconfig 打印网卡信息
reboot 重启
tar zxvf 文件名
cp [-r]原文件 新文件路径 :复制文件
-r 复制文件夹
ctrl+c 中断
mv 原文件 新文件 :剪切(具备重命名的功能)
rm [-rf] 文件名 :删除
-r 删除目录
-f 强制删除，不需要确认
clear 清屏

3.linux所有需要写路径的地方都支持两种写法
全路径 以/开头
相对路径 从当前资源一步一步寻找其他资源

4.Xshell
linux的客户端工具
CentOS默认只开启22端口，其他端口都被防火墙拦截

5.Filezilla工具使用
FTP协议
file transfer protocol
作用：互联网中，计算机之间直接传输文件需要遵循的协议
filezilla
基于FTP协议的客户端工具
使用filezilla可以快速完成文件的传输功能

6.安装jdk
把压缩包上传到服务器/usr/local/tmp
解压压缩包

    #tar zxvf 文件名
把解压后的文件复制到/usr/local/jdk7
    #cp -r 文件名 /usr/local/jdk7
配置环境变量，执行命令
    #vim /etc/profile
    修改文件中内容
    (分割使用的是:)
    ($变量名是引用的意思)
    (export 添加或者修改系统变量)
{
#export PATH USER LOGNAME MAIL HOSTNAME HISTSIZE HISTCONTROL
 export JAVA_HOME=/usr/local/jdk7
 export PATH=$JAVA_HOME/bin:$PATH
 export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
 }
解析文件或重启系统
    #source /etc/profile
验证配置是否成功
    #java -version

7.mysql安装
上传解压
复制到/usr/local/mysql

    #cp -r mysql-5.6.31 /usr/local/mysql
进入mysql文件夹
    #cd /usr/local/mysql
添加用户组，命名为mysql
    #groupadd mysql
创建用户mysql，并指定所属群组mysql
    #useradd -r -g mysql mysql
变更mysql用户组有操作当前文件夹的权限(必须保证在/usr/local/mysql目录下)

    #chown -R mysql .
初始化
判断/etc/my.cnf是否存在，存在就删除
    #ls /etc/my.cnf
    #rm /etc/my.cnf    
初始化数据库
    #./scripts/mysql_install_db --user=mysql 
修改配置文件
    复制my.cnf
    #cp support-files/my-default.cnf /etc/my.cnf 
    复制启动文件
    #cp support-files/mysql.server /etc/rc.d/init.d/mysql 
如果提示没有mysql命令，需要添加软连接
    #ln -s /usr/local/mysql/bin/mysql /usr/bin/mysql 
启动数据库
    #service mysql start 
操作数据库
    #mysql -u root -p

一、运行程序之前
1.Path 与classpath
Path顺序：在任意目录下执行Java工具文件–>找到直接执行Java工具文件
没找到的话–>把Java工具路径（JDK路径）的位置添加到系统的环境变量Path中
classPath:当我们需要一个class的执行文件时，系统就会在classpath中查找，找不到报错

二、基础知识
1.主方法里面的args是什么意思？
String[] args:是保存运行main函数时输入的参数的字符串数组
2.& 和&&的区别
他们都可以进行逻辑与的运算，表达的逻辑是当运算符两边的表达式结果都为true时，运算结果为true，否则为false;
不同的是&&具有短路功能，当第一个表达式的结果为false则不执行第二个表达式
3.Java的数据类型
基本数据类型： 引用数据类型：
整数类型：byte 1字节 -128-127 类 接口 数组 字符串
short 2字节 int 4字节
long 8字节
浮点类型：float 4字节 double 8字节
字符类型：char 2字节
Boolean类型：占一位 值只有true false
4.==运算符
比较运算符，返回值boolean类型，如果比较基本数据类型，比较的是==两边的值是否相等；
如果比较的引用类型，则比较的是引用的地址，也就是引用变量指向的地址
比较字符串的内容用equals()方法
5.什么是实例变量？什么是类变量
1）类变量又叫静态变量，也就是在变量前面加了static，静态变量在创建对象时就已经存在于内存中，随类的创建而创建

6.创建MyBatis SqlSessionFactory 的几种方式

    //配置文件的名称    主配置文件
    String resource = "ssm/configuration.xml";
    //通过Mybatis包中的Resources对象获取到配置文件
    Reader reader = Resources.getResourceAsReader(resource);
    //通过SqlSessionFactoryBuilder创建
    SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(reader);
    //获得session实例
    SqlSession session =sqlSessionFactory.openSession();
    User user = new User();
    user.setId(8);
    //完成数据库的插入
    session.insert("add", user);
    session.commit();
    session.close();
    System.out.println(sqlSessionFactory);

    通过Mapper动态代理的方式    用来替代Dao层
    String resource = "sqlMapConfig.xml";
    Reader rd = Resources.getResourceAsReader(resource);
    SqlSessionFactory ssf = new SqlSessionFactoryBuilder().build(rd);
    SqlSession session = ssf.openSession();
    UserMapper mapper = session.getMapper(UserMapper.class);
    mapper.deleteUserById();
    mapper.selectALT+/;
    mapper.insertUser(user);
    mapper.updateUser(user);
    session.commit();
     * Mapper动态代理四个原则
     * 1.接口方法名需要与Mapper.xml中要调用的SQL语句ID一致 
     * 2.接口的形参类型需要与Mapper.xml中的 ParameterType一致    
     * 3.接口的返回值类型需要与Mapper.xml中的返回值类型一致
     * 4.Mapper.xml中的NameSpace与接口的全包名一致
     * 5.注意事项：Mapper动态代理开发中，select类型根据返回值类型自动选择

8.MyBatis主配置文件sqlMapConfig.xml说明注意事项：
主配置文件按照以下顺序读取配置，顺序不对报错
    1.properties（读取配置文件） 用来连接数据库 需要建一个properties文件！
    2.settings（全局配置参数）
    3.typeAliases(类型别名)用来给mapper.xml文件中返回值的完整包名起一个缩写的别名！
    <typeAliases>
    <typeAlias type="com.yt.bean.User" alias="User"/>
    </typeAliases>
    4.typeHandlers(类型处理器)
    5.objectFactory(对象工厂)
    6.plugins(插件)
    7.environments(环境集合属性对象    Spring框架整合后不需要)
    8.mappers (配置映射器位置)推荐包的形式,mapper.xml文件应与 Mapper接口名字一样且在一个包下！

10.关联查询
    左关联：
    SELECT 
    u.id,u.username,u.sex,r.idroles,r.rolename <!-- 要查找的属性--!>
    FROM users u                                <!-- 来自那个表--!>
    LEFT JOIN roles r                            <!-- 关联那个表--!>
    ON u.id=r.idroles                           <!-- 关联的字段主键 --!>
    一对多查询
    SELECT 
    u.id,u.username,u.sex,r.idroles,r.rolename <!-- 要查找的属性--!>
    FROM users u                                <!-- 来自一的表--!>
    LEFT JOIN roles r                            <!-- 关联多的表--!>
    ON u.id=r.idroles                           <!-- 关联的字段主键 --!>
